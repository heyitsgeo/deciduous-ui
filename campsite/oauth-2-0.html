OAuth 2.0

Badge Brief

OAuth 2.0

Requirements

  1. Describe the origins of OAuth and the problem that it is intended to solve.
  2. Describe the difference between OAuth 1.x and OAuth 2.0.
  3. Describe the difference between OAuth 2.0 and Open ID Connect (OIDC).
  4. Identify several OAuth 2.0 or OIDC providers (SaaS and on-premise, etc.)
  5. Set up an account with an OAuth 2.0 provider, such as Auth0.
  6. Identify the five default grant types for OAuth 2.0.
    1. Identify use cases for each grant type.
  7. Build an OAuth 2.0 client application that uses the Authorization Code grant type. Connect this to the Identity Provider you set up previously. Demonstrate to your Guide that you can log in to your client application using the OAuth 2.0 provider.
  8. Describe the difference between opaque (reference) tokens and transparent tokens, such as JWTs. What are the pros and cons of each?
  9. Describe security threats that can face OAuth 2.0 implementations and how to mitigate those threats.
  10. Build an OAuth 2.0 provider from scratch and implement the following grant types: * Authorization Code * Refresh Token * Password
  11. Update your OAuth 2.0 client application to use your OAuth 2.0 provider instead of the SaaS or on-premise provider it originally used. Demonstrate to your Guide that your client and provider apps integrate properly.

Constraints

  • When building your OAuth 2.0 client and provider, you may not use any OAuth SDKs, frameworks, libraries, or middleware. You must write your own implementations.
  • You may (and should) use SDKs for performing any hashing, encryption, or JWT signing and verifying.
  • You may use frameworks within your OAuth 2.0 provider for managing logged-in user state in cookies or sessions. e.g., Microsoft Identity.
